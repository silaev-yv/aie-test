# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

Расширено в рамках HW03 с дополнительными эвристиками качества данных и параметрами CLI.

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.

#### Новые параметры команды report (HW03)

- `--max-hist-columns N` – максимальное количество числовых колонок для гистограмм (по умолчанию 6)
- `--top-k-categories N` – количество top-значений для категориальных признаков (по умолчанию 5)
- `--title "Текст"` – заголовок отчёта (по умолчанию "EDA-отчёт")
- `--min-missing-share 0.3` – порог доли пропусков, выше которого колонка считается проблемной (по умолчанию 0.3)

#### Примеры использования с новыми параметрами

```bash
# Отчёт с пользовательским заголовком
uv run eda-cli report data/example.csv --out-dir reports --title "Анализ данных продаж"

# Отчёт с большим количеством top-категорий и гистограмм
uv run eda-cli report data/example.csv --out-dir reports --top-k-categories 10 --max-hist-columns 10

# Отчёт с более строгим порогом проблемных пропусков
uv run eda-cli report data/example.csv --out-dir reports --min-missing-share 0.2

# Комбинация нескольких параметров
uv run eda-cli report data/example.csv \
    --out-dir reports_custom \
    --title "Детальный анализ" \
    --top-k-categories 8 \
    --max-hist-columns 12 \
    --min-missing-share 0.15
```

## Новые эвристики качества данных (HW03)

Помимо базовых проверок, теперь добавлены:

1. **has_constant_columns** – выявление колонок, где все значения одинаковые
2. **has_high_cardinality_categoricals** – обнаружение категориальных признаков с очень большим числом уникальных значений (>90% от количества строк)
3. **has_many_zero_values** – проверка числовых колонок на большую долю нулевых значений (>50%)
4. **has_suspicious_id_duplicates** – проверка ID-подобных колонок на наличие дубликатов

Все эти эвристики учитываются в итоговом `quality_score` и отображаются в отчёте.
